# Release and Acceptance Model

Acceptance identifies:

- exact source commit;
- exact predecessor;
- gate and environment;
- correctness result;
- evidence hashes;
- warnings and non-claims;
- accepted tag or release identity.

Release adds:

- clean build;
- SBOM;
- artifact hashes;
- provenance;
- signing or attestation where applicable;
- installation and rollback identity.
