# ISRAS 0.1.6 Release Notes

**Release date:** 2026-07-21

**Assurance status:** CORRECTIVE RELEASE CANDIDATE — SELF-VALIDATION REQUIRED

**Release tag:** `isras-v0.1.6`

## Purpose

ISRAS 0.1.6 preserves the complete Workstream A engineering baseline from 0.1.5
and corrects a release-artifact production defect discovered before 0.1.5
publication.

## Corrected Go release boundary

The release-artifact producer now treats the exact patch-level `go.mod` `go`
directive as a minimum:

- the exact minimum remains accepted;
- later compatible patch and minor releases are accepted;
- valid custom-suffix toolchains such as `go1.26.5-X:nodwarf5` are accepted;
- invalid version identities fail closed; and
- selected toolchains below the declared minimum fail closed.

Provenance records the actual selected Go toolchain. A byte-for-byte
reproducibility claim requires the same selected toolchain, exact source commit,
signed tag, file lists, producer implementation, and explicit provenance inputs.

## 0.1.5 disposition

`isras-v0.1.5` remains an immutable signed annotated tag. It was not published,
has no canonical release assets or GitHub Release, and is not consumer adoption
authority. It is not moved, deleted, or replaced.

## Exact release assets

A completed 0.1.6 release contains exactly:

```text
isras-validator-linux-amd64
isras-project-framework.tar.gz
isras-contracts.tar.gz
provenance.json
SHA256SUMS
SHA512SUMS
```

## Release completion

Release completion requires the exact signed corrective commit, hosted
validation, governed merge to `dev`, clean-clone release validation, signed
annotated tag, deterministic six-asset production and reproduction, read-only
publication preflight, draft-first controlled publication, remote-byte
verification, and final post-publication verification.

## Adoption readiness

Consumer adoption remains separate. A consuming repository may pin 0.1.6 only
after the signed tag and verified six-asset GitHub Release exist and that
repository passes its own adoption or upgrade validation.

## Limitations

- The release remains self-authored and self-validated.
- No independent audit, certification, regulatory approval, or universal
  production-fitness claim is made.
- The validator release artifact remains linux/amd64.
