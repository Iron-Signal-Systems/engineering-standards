# ISRAS 0.1.8 Release Notes

**Release date:** 2026-07-22

**Assurance status:** SIGNER-ROTATION RELEASE CANDIDATE — SELF-VALIDATION REQUIRED

**Release tag:** `isras-v0.1.8`

## Purpose

ISRAS 0.1.8 adds the Arch development host SSH signing key to the governed
hosted signer inventory. This permits pinned hosted validation to verify commits
signed by the new key without weakening exact principal and fingerprint checks.

## Security boundary

The release preserves the fail-closed signer trust model. The allowed-signers
bytes, SHA-256 declaration, and manifest must agree exactly. A consuming
repository cannot select or extend the keys trusted to verify its own commit.

ISRAS 0.1.7 and all earlier tags and release assets remain immutable. Consumers
must explicitly upgrade to 0.1.8 before the new signer is accepted.

## Exact release assets

The release contains exactly the governed validator, project framework,
contracts archive, provenance, and SHA-256 and SHA-512 manifests.

## Limitations

- The release remains self-authored and self-validated.
- No independent audit or certification is claimed.
- Publication and consumer adoption remain separate acceptance boundaries.
